### 2. 智能控制台输出

```typescript
// src/logger/terminal.ts
function detectTerminalCapabilities(): {
  supportsColor: boolean;
  supportsFormat: boolean;
} {
  const supportsColor = process.stdout.isTTY &&
    (process.env.TERM !== "dumb" ||
     process.env.COLORTERM !== "");

  const supportsFormat = supportsColor &&
    (process.env.TERM_PROGRAM === "iTerm.app" ||
     process.env.TERM === "xterm-256color");

  return { supportsColor, supportsFormat };
}
```

**Leon点评**：智能终端检测让日志输出更友好：
1. **颜色支持**：检测终端是否支持 ANSI 颜色
2. **格式检测**：检测终端是否支持高级格式化
3. **回退机制**：不支持时回退到纯文本
4. **环境变量**：支持通过环境变量覆盖检测

### 3. 文件滚动管理

```typescript
// src/logger/file-transport.ts
export class RotatingFileTransport {
  private currentStream: fs.WriteStream;
  private currentDate: string;

  async write(message: string): Promise<void> {
    const date = new Date().toISOString().split("T")[0];

    // 日期变更，创建新文件
    if (date !== this.currentDate) {
      await this.rotateFile(date);
    }

    this.currentStream.write(message);
  }

  private async rotateFile(date: string): Promise<void> {
    this.currentStream?.end();

    const filename = `openclaw-${date}.log`;
    this.currentStream = fs.createWriteStream(filename, { flags: "a" });
    this.currentDate = date;
  }
}
```

### 异步写入

// 使用异步流避免阻塞
const logStream = fs.createWriteStream(filename, { flags: "a" });

// 批量写入缓冲
const writeBuffer: string[] = [];
let flushTimer: NodeJS.Timeout | null = null;

function scheduleFlush() {
  if (flushTimer) return;
  flushTimer = setTimeout(() => {
    logStream.write(writeBuffer.join("\n"));
    writeBuffer.length = 0;
    flushTimer = null;
  }, 100);  // 100ms 批处理窗口
}

## 四、技术权衡

### 1. 文件滚动 vs 单文件

| 方案 | 优势 | 劣势 |
|------|------|------|
| 文件滚动 | 易管理、可搜索 | 复杂度高 |
| 单文件 | 简单、直接 | 难以管理大文件 |

**选择**：按日期滚动
**原因**：每天的日志独立，便于归档和分析

### 2. 同步 vs 异步写入

| 方案 | 优势 | 劣势 |
|------|------|------|
| 同步写入 | 简单、保证写入 | 阻塞主线程 |
| 异步写入 | 非阻塞、高性能 | 可能丢失日志 |

**选择**：异步写入 + 批处理
**原因**：性能优先，批处理减少系统调用

### 3. 彩色 vs 纯文本

| 方案 | 优势 | 劣势 |
|------|------|------|
| 彩色 | 可读性强、易识别 | 终端不支持时混乱 |
| 纯文本 | 兼容性好 | 可读性差 |

**选择**：智能检测
**原因**：自动检测终端能力，回退到纯文本
