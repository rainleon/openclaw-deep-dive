# 自动回复引擎 (Auto-Reply Engine)

> "OpenClaw的自动回复引擎设计得太精妙了。指令解析系统支持5种级别的细粒度控制（thinking、verbose、elevated、reasoning、notice），而且每个指令都可以在运行时动态调整。卧槽，这个队列系统的三种丢弃策略（summarize、drop、merge）考虑得太周到了，特别是 'summarize' 会自动将丢弃的后续内容合并成摘要。"

---

## 核心技术洞察

### 1. 多级指令解析：正则匹配 + 参数提取

```typescript
// src/auto-reply/reply/directives.ts
const matchLevelDirective = (
  body: string,
  names: string[],
): { start: number; end: number; rawLevel?: string } | null => {
  const namePattern = names.map(escapeRegExp).join("|");
  // 匹配 /name 或 /name:level 格式
  const match = body.match(new RegExp(`(?:^|\\s)\\/(?:${namePattern})(?=$|\\s|:)`, "i"));
  if (!match || match.index === undefined) {
    return null;
  }
  const start = match.index;
  let end = match.index + match[0].length;

  // 跳过空格
  let i = end;
  while (i < body.length && /\s/.test(body[i])) {
    i += 1;
  }

  // 检查是否有 : 分隔符
  if (body[i] === ":") {
    i += 1;
    while (i < body.length && /\s/.test(body[i])) {
      i += 1;
    }
  }

  // 提取级别参数（字母和连字符）
  const argStart = i;
  while (i < body.length && /[A-Za-z-]/.test(body[i])) {
    i += 1;
  }
  const rawLevel = i > argStart ? body.slice(argStart, i) : undefined;
  end = i;

  return { start, end, rawLevel };
};

const extractLevelDirective = <T>(
  body: string,
  names: string[],
  normalize: (raw?: string) => T | undefined,
): ExtractedLevel<T> => {
  const match = matchLevelDirective(body, names);
  if (!match) {
    return { cleaned: body.trim(), hasDirective: false };
  }
  const rawLevel = match.rawLevel;
  const level = normalize(rawLevel);  // 规范化级别值

  // 移除指令，保持剩余文本
  const cleaned = body
    .slice(0, match.start)
    .concat(" ")
    .concat(body.slice(match.end))
    .replace(/\s+/g, " ")
    .trim();

  return {
    cleaned,
    level,
    rawLevel,
    hasDirective: true,
  };
};

// 使用示例
export function extractThinkDirective(body?: string): {
  cleaned: string;
  thinkLevel?: ThinkLevel;
  rawLevel?: string;
  hasDirective: boolean;
} {
  if (!body) {
    return { cleaned: "", hasDirective: false };
  }
  const extracted = extractLevelDirective(
    body,
    ["thinking", "think", "t"],  // 支持的别名
    normalizeThinkLevel
  );
  return {
    cleaned: extracted.cleaned,
    thinkLevel: extracted.level,
    rawLevel: extracted.rawLevel,
    hasDirective: extracted.hasDirective,
  };
}
```

**Leon点评**：这个指令解析系统设计得很灵活：
1. **多别名支持**：每个指令支持多个名称（如 `thinking`、`think`、`t`）
2. **可选级别参数**：支持 `/thinking:low` 这样的语法
3. **大小写不敏感**：使用 `/i` 标志
4. **文本清理**：移除指令后保持剩余文本的格式
5. **类型安全**：每个指令都有对应的 TypeScript 类型

这种设计比简单的字符串匹配更强大，同时保持了代码的可维护性。

### 2. 队列系统：三种丢弃策略

```typescript
// src/auto-reply/reply/queue/state.ts
export type FollowupQueueState = {
  items: FollowupRun[];
  draining: boolean;
  lastEnqueuedAt: number;
  mode: QueueMode;           // "auto" | "manual"
  debounceMs: number;        // 防抖延迟
  cap: number;               // 队列容量
  dropPolicy: QueueDropPolicy; // "summarize" | "drop" | "merge"
  droppedCount: number;
  summaryLines: string[];    // 丢弃内容的摘要
  lastRun?: FollowupRun["run"];
};

export const DEFAULT_QUEUE_DEBOUNCE_MS = 1000;
export const DEFAULT_QUEUE_CAP = 20;
export const DEFAULT_QUEUE_DROP: QueueDropPolicy = "summarize";

// 全局队列存储（按 key 分隔）
export const FOLLOWUP_QUEUES = new Map<string, FollowupQueueState>();

export function getFollowupQueue(
  key: string,
  settings: QueueSettings,
): FollowupQueueState {
  const existing = FOLLOWUP_QUEUES.get(key);
  if (existing) {
    // 应用运行时设置更新
    applyQueueRuntimeSettings({
      target: existing,
      settings,
    });
    return existing;
  }

  // 创建新队列
  const created: FollowupQueueState = {
    items: [],
    draining: false,
    lastEnqueuedAt: 0,
    mode: settings.mode,
    debounceMs: settings.debounceMs ?? DEFAULT_QUEUE_DEBOUNCE_MS,
    cap: settings.cap ?? DEFAULT_QUEUE_CAP,
    dropPolicy: settings.dropPolicy ?? DEFAULT_QUEUE_DROP,
    droppedCount: 0,
    summaryLines: [],
  };

  FOLLOWUP_QUEUES.set(key, created);
  return created;
}
```

**Leon点评**：队列系统设计得很周到：
1. **三种丢弃策略**：
   - `summarize`：将丢弃的内容添加到摘要中
   - `drop`：直接丢弃
   - `merge`：合并到下一个项目
2. **防抖机制**：避免频繁触发（默认 1000ms）
3. **容量限制**：防止无限增长（默认 20）
4. **全局 Map 存储**：按 key 分隔不同会话的队列
5. **运行时设置更新**：支持动态调整队列行为

这种设计让 OpenClaw 能够处理消息洪峰，同时保持响应的及时性。

### 3. 队列指令解析：复杂参数解析

```typescript
// src/auto-reply/reply/queue/directive.ts
function parseQueueDirectiveArgs(raw: string): {
  consumed: number;
  queueMode?: QueueMode;
  queueReset: boolean;
  rawMode?: string;
  debounceMs?: number;
  cap?: number;
  dropPolicy?: QueueDropPolicy;
  rawDebounce?: string;
  rawCap?: string;
  rawDrop?: string;
  hasOptions: boolean;
} {
  const len = raw.length;
  let i = skipDirectiveArgPrefix(raw);  // 跳过前导空格和 ":"
  let consumed = i;
  let queueMode: QueueMode | undefined;
  let queueReset = false;
  let hasOptions = false;

  const takeToken = (): string | null => {
    const res = takeDirectiveToken(raw, i);
    i = res.nextIndex;
    return res.token;
  };

  while (i < len) {
    const token = takeToken();
    if (!token) {
      break;
    }
    const lowered = token.trim().toLowerCase();

    // 重置命令
    if (lowered === "default" || lowered === "reset" || lowered === "clear") {
      queueReset = true;
      consumed = i;
      break;
    }

    // 防抖延迟: debounce:500ms 或 debounce=500ms
    if (lowered.startsWith("debounce:") || lowered.startsWith("debounce=")) {
      const rawDebounce = token.split(/[:=]/)[1] ?? "";
      debounceMs = parseQueueDebounce(rawDebounce);
      hasOptions = true;
      consumed = i;
      continue;
    }

    // 容量限制: cap:10 或 cap=10
    if (lowered.startsWith("cap:") || lowered.startsWith("cap=")) {
      const rawCap = token.split(/[:=]/)[1] ?? "";
      cap = parseQueueCap(rawCap);
      hasOptions = true;
      consumed = i;
      continue;
    }

    // 丢弃策略: drop:summarize 或 drop=summarize
    if (lowered.startsWith("drop:") || lowered.startsWith("drop=")) {
      const rawDrop = token.split(/[:=]/)[1] ?? "";
      dropPolicy = normalizeQueueDropPolicy(rawDrop);
      hasOptions = true;
      consumed = i;
      continue;
    }

    // 队列模式: auto 或 manual
    const mode = normalizeQueueMode(token);
    if (mode) {
      queueMode = mode;
      consumed = i;
      continue;
    }

    // 遇到无法识别的 token，停止解析
    break;
  }

  return {
    consumed,
    queueMode,
    queueReset,
    debounceMs,
    cap,
    dropPolicy,
    rawDebounce,
    rawCap,
    rawDrop,
    hasOptions,
  };
}

export function extractQueueDirective(body?: string): {
  cleaned: string;
  queueMode?: QueueMode;
  queueReset: boolean;
  hasDirective: boolean;
  // ... 其他字段
} {
  if (!body) {
    return {
      cleaned: "",
      hasDirective: false,
      queueReset: false,
      hasOptions: false,
    };
  }

  // 匹配 /queue 指令
  const re = /(?:^|\s)\/queue(?=$|\s|:)/i;
  const match = re.exec(body);
  if (!match) {
    return {
      cleaned: body.trim(),
      hasDirective: false,
      queueReset: false,
      hasOptions: false,
    };
  }

  const start = match.index + match[0].indexOf("/queue");
  const argsStart = start + "/queue".length;
  const args = body.slice(argsStart);
  const parsed = parseQueueDirectiveArgs(args);

  // 移除指令，保留其他内容
  const cleanedRaw = `${body.slice(0, start)} ${body.slice(argsStart + parsed.consumed)}`;
  const cleaned = cleanedRaw.replace(/\s+/g, " ").trim();

  return {
    cleaned,
    queueMode: parsed.queueMode,
    queueReset: parsed.queueReset,
    debounceMs: parsed.debounceMs,
    cap: parsed.cap,
    dropPolicy: parsed.dropPolicy,
    hasDirective: true,
    hasOptions: parsed.hasOptions,
  };
}
```

**Leon点评**：队列指令解析器支持多种语法：
1. **模式设置**：`/queue auto` 或 `/queue manual`
2. **防抖配置**：`/queue debounce:500ms` 或 `/queue debounce=500ms`
3. **容量限制**：`/queue cap:10` 或 `/queue cap=10`
4. **丢弃策略**：`/queue drop:summarize` 或 `/queue drop=merge`
5. **重置**：`/queue reset` 或 `/queue default`

这种设计让用户可以在运行时精确控制队列行为，而且支持 `:` 和 `=` 两种分隔符，更灵活。

### 4. 命令检测系统：粗检测 + 精验证

```typescript
// src/auto-reply/command-detection.ts
export function hasControlCommand(
  text?: string,
  cfg?: OpenClawConfig,
  options?: CommandNormalizeOptions,
): boolean {
  if (!text) {
    return false;
  }
  const trimmed = text.trim();
  if (!trimmed) {
    return false;
  }

  // 规范化命令体（处理引号、前缀等）
  const normalizedBody = normalizeCommandBody(trimmed, options);
  if (!normalizedBody) {
    return false;
  }

  const lowered = normalizedBody.toLowerCase();

  // 获取所有已注册的命令
  const commands = cfg ? listChatCommandsForConfig(cfg) : listChatCommands();

  for (const command of commands) {
    for (const alias of command.textAliases) {
      const normalized = alias.trim().toLowerCase();
      if (!normalized) {
        continue;
      }

      // 完全匹配
      if (lowered === normalized) {
        return true;
      }

      // 前缀匹配（如果命令接受参数）
      if (command.acceptsArgs && lowered.startsWith(normalized)) {
        const nextChar = normalizedBody.charAt(normalized.length);
        if (nextChar && /\s/.test(nextChar)) {
          return true;
        }
      }
    }
  }

  return false;
}

// 粗略检测内联指令（用于优化）
export function hasInlineCommandTokens(text?: string): boolean {
  const body = text ?? "";
  if (!body.trim()) {
    return false;
  }
  // 检测 / 或 ! 开头的命令
  return /(?:^|\s)[/!][a-z]/i.test(body);
}

// 组合检测
export function shouldComputeCommandAuthorized(
  text?: string,
  cfg?: OpenClawConfig,
  options?: CommandNormalizeOptions,
): boolean {
  return isControlCommandMessage(text, cfg, options) || hasInlineCommandTokens(text);
}
```

**Leon点评**：这个命令检测系统有两层：
1. **粗检测**（`hasInlineCommandTokens`）：快速检测是否有 `/` 或 `!` 开头的命令，用于优化
2. **精验证**（`hasControlCommand`）：精确匹配已注册的命令列表，支持别名和参数

这种设计在性能和准确性之间取得了平衡。粗检测可以快速过滤非命令消息，精验证确保只有真实命令才会触发处理。

### 5. 发送策略控制：/send 命令

```typescript
// src/auto-reply/send-policy.ts
export type SendPolicyOverride = "allow" | "deny";

export function normalizeSendPolicyOverride(raw?: string | null): SendPolicyOverride | undefined {
  const value = raw?.trim().toLowerCase();
  if (!value) {
    return undefined;
  }
  if (value === "allow" || value === "on") {
    return "allow";
  }
  if (value === "deny" || value === "off") {
    return "deny";
  }
  return undefined;
}

export function parseSendPolicyCommand(raw?: string): {
  hasCommand: boolean;
  mode?: SendPolicyOverride | "inherit";
} {
  if (!raw) {
    return { hasCommand: false };
  }

  const trimmed = raw.trim();
  if (!trimmed) {
    return { hasCommand: false };
  }

  const normalized = normalizeCommandBody(trimmed);
  const match = normalized.match(/^\/send(?:\s+([a-zA-Z]+))?\s*$/i);

  if (!match) {
    return { hasCommand: false };
  }

  const token = match[1]?.trim().toLowerCase();
  if (!token) {
    return { hasCommand: true };  // /send 无参数
  }

  if (token === "inherit" || token === "default" || token === "reset") {
    return { hasCommand: true, mode: "inherit" };
  }

  const mode = normalizeSendPolicyOverride(token);
  return { hasCommand: true, mode };
}
```

**Leon点评**：`/send` 命令提供了细粒度的发送控制：
1. **无参数**：`/send` - 只是标记为发送命令（不改变策略）
2. **允许**：`/send allow` 或 `/send on` - 强制允许发送
3. **拒绝**：`/send deny` 或 `/send off` - 强制拒绝发送
4. **继承**：`/send inherit` 或 `/send reset` - 恢复默认策略

这种设计让用户可以在单次对话中临时覆盖全局发送策略，非常灵活。

### 6. 消息分发流程：完整处理管道

```typescript
// src/auto-reply/dispatch.ts
export async function dispatchInboundMessage(params: {
  ctx: MsgContext | FinalizedMsgContext;
  cfg: OpenClawConfig;
  dispatcher: ReplyDispatcher;
  replyOptions?: Omit<GetReplyOptions, "onToolResult" | "onBlockReply">;
  replyResolver?: typeof import("./reply.js").getReplyFromConfig;
}): Promise<DispatchInboundResult> {
  // 1. 规范化入站上下文
  const finalized = finalizeInboundContext(params.ctx);

  // 2. 使用 dispatcher 包装执行
  return await withReplyDispatcher({
    dispatcher: params.dispatcher,
    run: () =>
      dispatchReplyFromConfig({
        ctx: finalized,
        cfg: params.cfg,
        dispatcher: params.dispatcher,
        replyOptions: params.replyOptions,
        replyResolver: params.replyResolver,
      }),
  });
}

// dispatcher 包装器确保资源清理
export async function withReplyDispatcher<T>(params: {
  dispatcher: ReplyDispatcher;
  run: () => Promise<T>;
  onSettled?: () => void | Promise<void>;
}): Promise<T> {
  try {
    return await params.run();
  } finally {
    // 确保在所有退出路径上释放 dispatcher 预留
    params.dispatcher.markComplete();
    try {
      await params.dispatcher.waitForIdle();
    } finally {
      await params.onSettled?.();
    }
  }
}
```

**Leon点评**：消息分发流程设计得很严谨：
1. **上下文规范化**：`finalizeInboundContext` 确保所有必需字段都存在
2. **Dispatcher 包装**：`withReplyDispatcher` 确保资源正确释放
3. **错误隔离**：即使处理失败，也会清理预留的资源
4. **可配置解析器**：`replyResolver` 参数允许注入自定义逻辑

这种设计避免了资源泄漏，同时保持了代码的灵活性。

## 三、队列系统详解

### 队列状态结构

```typescript
type FollowupQueueState = {
```

### 丢弃策略详解

| 策略 | 行为 | 适用场景 |
|------|------|----------|
| `summarize` | 将丢弃的内容添加到摘要中 | 消息洪峰，需要保留上下文 |
| `drop` | 直接丢弃，不保留 | 低优先级消息，无需保留 |
| `merge` | 合并到下一个项目 | 相关消息，可以一起处理 |

## 五、关键技术权衡

### 1. 正则匹配 vs 解析器

| 方案 | 优势 | 劣势 |
|------|------|------|
| 正则匹配 | 简单、快速 | 复杂语法难以处理 |
| 解析器 | 强大、可扩展 | 复杂、性能较低 |

**选择**：正则匹配
**原因**：指令语法简单，正则匹配足够且性能更好

### 2. 全局队列 Map vs 会话本地队列

| 方案 | 优势 | 劣势 |
|------|------|------|
| 全局 Map | 跨会话共享、简单 | 需要手动管理生命周期 |
| 会话本地 | 自动清理 | 难以跨会话访问 |

**选择**：全局 Map
**原因**：支持跨会话队列，便于调试和管理

### 3. 粗检测 vs 直接精验证

| 方案 | 优势 | 劣势 |
|------|------|------|
| 粗检测 + 精验证 | 性能优化、减少不必要处理 | 两层检测，代码稍多 |
| 直接精验证 | 简单 | 所有消息都要遍历命令列表 |

**选择**：粗检测 + 精验证
**原因**：大多数消息不是命令，粗检测可以快速过滤

### 4. 指令文本清理 vs 原样保留

| 方案 | 优势 | 劣势 |
|------|------|------|
| 清理指令 | 避免指令污染 LLM 上下文 | 需要额外处理 |
| 原样保留 | 简单 | 指令可能干扰 LLM |

**选择**：清理指令
**原因**：指令是给 OpenClaw 看的，不是给 LLM 看的

## 六、自动回复引擎与其他模块的关系

**Q：指令解析和命令解析有什么区别？**

A：指令（directives）是控制 OpenClaw 行为的元命令（如 `/thinking:high`），命令（commands）是用户请求的功能（如 `/new`、`/config`）。指令被移除后不会发送给 LLM，命令可能会触发特定功能。

**Q：队列的 debounce 是如何工作的？**

A：当新消息到达时，如果距离上次入队时间小于 `debounceMs`，则更新最后一个项目而不是创建新项目。这可以避免在快速连续消息时创建过多队列项目。

**Q：三种丢弃策略的区别是什么？**

A：
- `summarize`：丢弃的内容会被添加到 `summaryLines`，在下次运行时作为上下文
- `drop`：直接丢弃，不保留任何信息
- `merge`：将丢弃的内容合并到下一个队列项目的消息中

**Q：发送策略覆盖是如何工作的？**

A：`/send` 命令设置一个会话级别的标志，优先于全局发送策略。`/send inherit` 恢复使用全局策略。这允许在单次对话中临时覆盖全局设置。

**Q：命令检测的性能如何？**

A：使用两层检测优化。粗检测使用简单的正则（`/[/!][a-z]/i`），可以快速过滤非命令消息。只有通过粗检测的消息才会进行精验证，遍历已注册命令列表。
