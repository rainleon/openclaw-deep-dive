# 移动端实现 (Mobile Implementations)

> "OpenClaw的移动端实现采用了混合架构——原生平台能力 + Node.js 运行时，通过 OpenClawKit 桥接层连接。iOS 基于 SwiftUI 构建响应式 UI，Android 使用 Jetpack Compose 实现 Material Design，macOS 则融合了 SwiftUI 和 AppKit。这种设计既保证了原生性能，又实现了跨平台代码复用。卧槽，这个后台任务优化太细致了——iOS 的 BGTaskScheduler、Android 的 ForegroundService、macOS 的 LaunchAgent，每个平台都利用了系统能力来保证可靠性。"

## 核心技术洞察

### 1. OpenClawKit 桥接层

```swift
// OpenClawKit (Swift 共享层)
public protocol OpenClawDelegate: AnyObject {
    func onMessageReceived(_ message: OpenClawMessage)
    func onGatewayConnected()
    func onGatewayDisconnected(error: Error?)
}

// JavaScript 桥接
@objc(OpenClawBridge)
public class OpenClawBridge: NSObject {
    private var nodeRuntime: NodeRuntime
    private weak var delegate: OpenClawDelegate?

    public func sendMessage(_ text: String, completion: @escaping (Bool) -> Void) {
        nodeRuntime.evaluateScript("sendMessage('\(text)')") { result in
            completion(result?.toBool() ?? false)
        }
    }
}
```

**Leon点评**：OpenClawKit 是整个移动端架构的核心：
1. **统一 API**：Swift 协议定义了所有平台共享的接口
2. **JavaScript 桥接**：通过 Node.js 运行时执行 OpenClaw 核心代码
3. **主线程安全**：大量使用 `@MainActor` 确保 UI 更新在主线程
4. **事件分发**：将 Node.js 事件转换为 Swift 回调

4. ** completion 处理**：正确报告任务完成状态

## 六、网关发现机制

### 多层次发现策略

```swift
// OpenClawKit/Core/GatewayDiscovery.swift
public enum GatewayDiscovery {
    public static func discover() async throws -> URL {
        // 1. 显式配置
        if let configuredUrl = loadConfiguredURL() {
            return configuredUrl
        }

        // 2. 本地网络扫描
        if let localUrl = await scanLocalNetwork() {
            return localUrl
        }

        // 3. Tailscale MagicDNS
        if let tailscaleUrl = await resolveTailscaleMagicDNS() {
            return tailscaleUrl
        }

        // 4. 云端网关
        if let cloudUrl = loadCloudURL() {
            return cloudUrl
        }

        throw GatewayDiscoveryError.notFound
    }
}
```

### WebSocket 连接管理

```swift
// OpenClawKit/Core/GatewayClient.swift
public class GatewayClient: NSObject, WebSocketDelegate {
    private var socket: WebSocket?
    private var isConnected = false

    public func connect(to url: URL) async throws {
        let request = URLRequest(url: url)
        socket = WebSocket(request: request)
        socket?.delegate = self
        socket?.resume()
    }

    public func didReceive(event: WebSocketEvent, client: WebSocket) {
        switch event {
        case .connected:
            isConnected = true
            delegate?.gatewayDidConnect()

        case .disconnected(let reason, let code):
            isConnected = false
            delegate?.gatewayDidDisconnect(reason: reason, code: code)

        case .text(let message):
            handleMessage(message)

        case .binary(let data):
            handleBinaryData(data)

        default:
            break
        }
    }
}
```

## 七、技术权衡

### 1. 原生 vs 跨平台框架

| 方案 | 优势 | 劣势 |
|------|------|------|
| 原生 | 最佳性能、完整 API | 开发成本高 |
| React Native | 跨平台、热更新 | 性能损失、包体积大 |

**选择**：原生
**原因**：需要深度集成系统功能（后台任务、推送通知）

### 2. 混合架构

| 组件 | 技术选型 |
|------|----------|
| UI | 原生 (SwiftUI / Jetpack Compose) |
| 逻辑 | OpenClawKit (Swift) + OpenClaw Core (JS) |
| 桥接 | Node.js Mobile |

**原因**：原生 UI 提供最佳体验，JS 核心实现代码复用

### 3. 后台任务策略

| 平台 | 策略 | 限制 |
|------|------|------|
| iOS | BGTaskScheduler | 最多 1 分钟 / 天 |
| Android | ForegroundService | 需要持久通知 |
| macOS | LaunchAgent | 无执行时间限制 |

**选择**：各平台使用最优策略
**原因**：系统限制不同，无法使用统一方案
