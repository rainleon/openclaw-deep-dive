## 核心技术洞察

### 1. 多层 MIME 检测策略的优先级设计

```typescript
// sniffed > extension > header，但通用类型不覆盖具体映射
if (sniffed && (!isGenericMime(sniffed) || !extMime)) {
  return sniffed;
}
if (extMime) {
  return extMime;
}
```

**Leon 的评价**：
这个优先级设计非常精妙。sniffed（魔法数字）最准确，但 ZIP 这种通用容器会误导判断（XLSX 会被识别成 ZIP），所以扩展名映射作为第二优先级可以纠正这个问题。卧槽，这种细节处理的真是到位。

### 2. 可选依赖的延迟加载模式

```typescript
let canvasModulePromise: Promise<CanvasModule> | null = null;
let pdfJsModulePromise: Promise<PdfJsModule> | null = null;

async function loadCanvasModule(): Promise<CanvasModule> {
  if (!canvasModulePromise) {
    canvasModulePromise = import("@napi-rs/canvas").catch((err) => {
      canvasModulePromise = null;
      throw new Error(`Optional dependency @napi-rs/canvas is required...`);
    });
  }
  return canvasModulePromise;

```

**Leon 的评价**：
这种模式太他妈关键了。不是所有人都需要 PDF 提取功能，不需要的人就不必安装这些重型依赖。按需加载 + 优雅降级，这才是插件化系统的正确姿势。

### 3. PDF 提取的智能降级策略


**Leon 的评价**：
作者把 SSRF 防护做成了基础设施层的能力，媒体管道直接复用。allowPrivateNetwork: false + hostnameAllowlist，这是默认拒绝、显式允许的安全哲学体现。这种设计比每个模块各自实现 SSRF 防护要靠谱得多。

### 5. 临时媒体服务的 TTL 单次清理

```typescript

// server.ts 中
```

**Leon 的评价**：
2 分钟的 TTL 设计得很巧妙：足够 AI 完成一次完整的媒体处理，又不会让垃圾文件堆积。服务器关闭时全量清理，这样设计简单可靠，不需要复杂的定时清理逻辑。不过如果 gateway 崩溃，可能会有残留，这是需要权衡的点。

### 6. HEIC 自动转 JPEG 的无缝处理


**Leon 的评价**：
这是为 Apple 生态用户做的贴心设计。iPhone 默认拍照格式是 HEIC，大多数 AI 模型不支持，自动转成 JPEG 对用户完全透明。这种细节能大幅降低 iOS 用户的使用摩擦，体现了作者对用户体验的关注。

### 7. 多维度限制的纵深防御


**Leon 的评价**：
这个限制清单太他妈全面了。文件大小、字符数、超时、重定向次数、MIME 类型白名单，每一层都是一道防线。特别是 PDF 的限制：maxPages 防止处理超大文档，maxPixels 防止内存爆炸，minTextChars 防止提取无意义内容。这种纵深防御思维是专业安全工程师的体现。

### 8. 文件名的安全防护

```typescript

// url: https://hostname/media/{id}
```

**Leon 的评价**：
UUID 命名 + 原始文件名保留，这个设计兼顾了安全性和可追溯性。URL 中只暴露 UUID，不泄露原始文件名；但本地存储保留原始名，方便调试。这种细节考虑很周到。

**智能降级流程**：
```typescript
// 1. 先提取文本（不需要 Canvas）
const textParts: string[] = [];
for (const pageNum of effectivePages) {
  const page = await pdf.getPage(pageNum);
  const textContent = await page.getTextContent();
  const pageText = textContent.items
    .map((item) => ("str" in item ? String(item.str) : ""))
    .filter(Boolean)
    .join(" ");
  if (pageText) {
    textParts.push(pageText);
  }
}

const text = textParts.join("\n\n");

// 2. 文本足够多，直接返回
if (text.trim().length >= minTextChars) {
  return { text, images: [] };
}

// 3. 文本不够，尝试提取图片
let canvasModule: CanvasModule;
try {
  canvasModule = await loadCanvasModule();
} catch (err) {
  onImageExtractionError?.(err);
  return { text, images: [] };  // 降级到纯文本
}

// 4. 渲染页面到 Canvas
const { createCanvas } = canvasModule;
const images: PdfExtractedImage[] = [];
const pixelBudget = Math.max(1, maxPixels);

for (const pageNum of effectivePages) {
  const page = await pdf.getPage(pageNum);
  const viewport = page.getViewport({ scale: 1 });
  const pagePixels = viewport.width * viewport.height;

  // 计算缩放比例，不超过像素预算
  const scale = Math.min(1, Math.sqrt(pixelBudget / Math.max(1, pagePixels)));
  const scaled = page.getViewport({ scale: Math.max(0.1, scale) });

  const canvas = createCanvas(Math.ceil(scaled.width), Math.ceil(scaled.height));
  await page.render({
    canvas: canvas as unknown as HTMLCanvasElement,
    viewport: scaled,
  }).promise;

  const png = canvas.toBuffer("image/png");
  images.push({
    type: "image",
    data: png.toString("base64"),
    mimeType: "image/png",
  });
}
```

**设计评价**：
- **先文本后图片**：文本提取是轻量级操作，先做这个可以快速返回
- **minTextChars 阈值**：避免对有足够文本的 PDF 做昂贵的图片渲染
- **像素预算控制**：scale 计算确保总像素不超过 maxPixels，防止内存爆炸
- **错误隔离**：Canvas 加载失败只影响图片提取，不影响文本

### 2.5 store.ts - 临时存储

**文件**：`src/media/store.ts`

**UUID 命名策略**：
```typescript
import { randomUUID } from "node:crypto";

export async function saveMediaSource(source: string): Promise<SavedMedia> {
  const id = randomUUID();
  const originalName = extractOriginalFilename(source); // 从路径提取
  const sanitizedName = sanitizeFilename(originalName);  // 移除危险字符

  const ext = path.extname(sanitizedName);
  const base = path.basename(sanitizedName, ext);
  const finalName = `${base}---${id}${ext}`;  // 原始名 + UUID

  const targetPath = path.join(mediaDir, finalName);

  // 复制文件到媒体目录
  await fs.copyFile(source, targetPath);

  const stats = await fs.stat(targetPath);
  if (stats.size > MAX_BYTES) {
    await fs.unlink(targetPath);
    throw new Error(`Media file too large: ${stats.size} > ${MAX_BYTES}`);
  }

  return { id: finalName, path: targetPath, size: stats.size };
}

const MAX_BYTES = 5 * 1024 * 1024;  // 5MB
```

**sanitizeFilename 实现**：
```typescript
function sanitizeFilename(fileName: string): string {
  // 移除路径遍历字符
  let sanitized = fileName.replace(/[\/\\]/g, "_");

  // 移除控制字符
  sanitized = sanitized.replace(/[\x00-\x1f\x80-\x9f]/g, "");

  // 移除首尾空格和点
  sanitized = sanitized.trim().replace(/^\.+|\.+$/g, "");

  // 长度限制
  const maxLength = 255;
  if (sanitized.length > maxLength) {
    const ext = path.extname(sanitized);
    const base = path.basename(sanitized, ext);
    sanitized = `${base.slice(0, maxLength - ext.length)}${ext}`;
  }

  return sanitized || "file";
}
```

### 2.6 server.ts - HTTP 服务

**文件**：`src/media/server.ts`

**核心设计**：
```typescript
export async function startMediaServer(
  port: number,
  ttlMs: number,
  runtime: RuntimeEnv,
): Promise<Server> {
  const mediaDir = getMediaStoreDir();
  await fs.mkdir(mediaDir, { recursive: true });

  const used = new Set<string>();  // 跟踪已服务的文件

  const server = createServer(async (req, res) => {
    if (!req.url?.startsWith("/media/")) {
      res.statusCode = 404;
      res.end("Not Found");
      return;
    }

    const id = decodeURIComponent(req.url.slice("/media/".length));
    const filePath = path.join(mediaDir, id);

    try {
      await fs.access(filePath);
      const stat = await fs.stat(filePath);
      const contentType = await detectMime({ filePath });
      const ext = path.extname(filePath);

      res.setHeader("Content-Type", contentType ?? "application/octet-stream");
      res.setHeader("Content-Length", stat.size);
      res.setHeader("Cache-Control", `max-age=${Math.floor(ttlMs / 1000)}`);

      const fileStream = createReadStream(filePath);
      fileStream.pipe(res);

      used.add(id);  // 标记为已使用

      // 发送完成后删除
      fileStream.on("end", () => {
        void fs.unlink(filePath).catch(() => {});
        used.delete(id);
      });
    } catch {
      res.statusCode = 404;
      res.end("Media Not Found");
    }
  });

  // TTL 清理定时器
  const cleanupTimer = setInterval(async () => {
    const now = Date.now();
    for (const file of await fs.readdir(mediaDir)) {
      const filePath = path.join(mediaDir, file);
      const stat = await fs.stat(filePath).catch(() => null);
      if (stat && now - stat.mtimeMs > ttlMs) {
        await fs.unlink(filePath).catch(() => {});
      }
    }
  }, ttlMs / 2);

  server.on("close", () => {
    clearInterval(cleanupTimer);
    // 清理所有剩余文件
    for (const id of used) {
      void fs.unlink(path.join(mediaDir, id)).catch(() => {});
    }
  });

  return new Promise((resolve) => {
    server.listen(port, () => resolve(server));
  });
}
```

**设计亮点**：
- **单次清理**：文件发送后立即删除，不依赖 TTL
- **TTL 后备**：定时器清理未访问的过期文件
- **优雅关闭**：服务器关闭时清理所有剩余文件

```typescript
export async function ensureMediaHosted(
  source: string,
  opts: { port?: number; startServer?: boolean; runtime?: RuntimeEnv } = {},
): Promise<HostedMedia> {
  const port = opts.port ?? DEFAULT_PORT;  // 42873
  const runtime = opts.runtime ?? defaultRuntime;

  const saved = await saveMediaSource(source);
  const hostname = await getTailnetHostname();  // Tailscale

  // 检查端口是否空闲
  const needsServerStart = await isPortFree(port);
  if (needsServerStart && !opts.startServer) {
    await fs.rm(saved.path).catch(() => {});
    throw new Error(
      `Media hosting requires the webhook/Funnel server. Start \`${formatCliCommand("openclaw webhook")}\`/\`${formatCliCommand("openclaw up")}\` or re-run with --serve-media.`,
    );
  }

  if (needsServerStart && opts.startServer) {
    if (!mediaServer) {
      mediaServer = await startMediaServer(port, TTL_MS, runtime);
      logInfo(
        `🦞 Started temporary media host on http://localhost:${port}/media/:id (TTL ${TTL_MS / 1000}s)`,
        runtime,
      );
      mediaServer.unref?.();  // 允许进程退出
    }
  }

  const url = `https://${hostname}/media/${saved.id}`;
  return { url, id: saved.id, size: saved.size };
}
```

```typescript
  const mime = normalizeMimeType(opts.contentType);
  if (mime && TELEGRAM_VOICE_MIME_TYPES.has(mime)) {
    return true;
  }

  const fileName = opts.fileName?.trim();
  if (!fileName) {
    return false;
  }

  const ext = getFileExtension(fileName);
  if (!ext) {
    return false;
  }

  return TELEGRAM_VOICE_AUDIO_EXTENSIONS.has(ext);
}
```

### 3.4 流式读取 + 超时检测

```typescript
// read-response-with-limit.ts
export async function readResponseWithLimit(
  res: Response,
  maxBytes: number,
  opts: { chunkTimeoutMs?: number; onOverflow?: (params: { maxBytes: number; res: Response }) => Error },
): Promise<Buffer> {
  const chunks: Buffer[] = [];
  let totalBytes = 0;
  let lastChunkTime = Date.now();

  const reader = res.body?.getReader();
  if (!reader) {
    throw new Error("Response body is not readable");
  }

  while (true) {
    const { done, value } = await reader.read();

    if (opts.chunkTimeoutMs) {
      const idleMs = Date.now() - lastChunkTime;
      if (idleMs > opts.chunkTimeoutMs) {
        throw new Error(`Response read timeout after ${idleMs}ms idle`);
      }
    }

    if (done) break;

    if (value) {
      lastChunkTime = Date.now();
      totalBytes += value.length;

      if (totalBytes > maxBytes) {
        throw opts.onOverflow?.({ maxBytes, res }) ||
          new Error(`Response exceeds ${maxBytes} bytes`);
      }

      chunks.push(Buffer.from(value));
    }
  }

  return Buffer.concat(chunks);
}
```

**设计亮点**：
- 分块读取，避免一次性加载大文件到内存
- 超时检测防止慢速攻击
- 回调自定义错误，保持灵活性

## 四、潜在问题与改进建议

### 4.1 媒体目录残留风险

**问题**：如果 gateway 崩溃（kill -9），media 目录的文件不会被清理。

**建议**：启动时扫描 media 目录，删除过期文件。

### 4.2 PDF 提取的像素预算分配

**问题**：`maxPixels` 是总预算，但多页 PDF 会按比例分配，可能每页都很小。

**当前逻辑**：
```typescript
const pixelBudget = Math.max(1, maxPixels);  // 总预算
const scale = Math.sqrt(pixelBudget / Math.max(1, pagePixels));  // 全局缩放
```

**建议**：改为按页分配预算，确保每页有最小分辨率。

### 4.3 HEIC 转换的性能开销

**问题**：每次请求都转换，没有缓存机制。

**建议**：对于相同内容的 HEIC 文件，缓存转换结果。

### 4.4 MIME 检测的魔法数字库更新

**问题**：file-type 库需要定期更新以支持新格式。

**建议**：定期检查依赖更新，或考虑使用系统原生的 file 命令。
